EKS Glossary

Quick reference for Amazon EKS terminology. Print-friendly.

Architecture

Control Plane AWS-managed
The Kubernetes brain — API server, etcd, scheduler, controller manager. Fully managed by AWS across 3 AZs. You never SSH into it.
Data Plane
The worker nodes that actually run your pods. Lives in your VPC. You manage it (unless using Fargate or EKS Auto Mode).
EKS-managed VPC
A separate VPC owned by AWS where the control plane runs. Connected to your VPC via cross-account ENIs.
Cross-account ENIs
Elastic Network Interfaces placed in your VPC's subnets that let the control plane communicate with your worker nodes.
Cluster Endpoint
The HTTPS URL for the Kubernetes API server. Can be public, private, or both. Workers and kubectl connect here.

Node Types

EKS Auto Mode AWS-managed
AWS manages everything — node provisioning, scaling, patching, networking. Least operational burden, least control.
Managed Node Group AWS-managed
AWS manages an Auto Scaling Group of EC2 instances for you. You choose instance type, AMI family, and scaling bounds. AWS handles rolling updates.
Self-Managed Node Group
You manage EC2 instances directly via your own Auto Scaling Group and launch template. Full control — required for custom AMIs that don't fit managed node group constraints.
Fargate AWS-managed
Serverless compute. Each pod gets its own isolated micro-VM. No nodes to manage. Limited to Linux, no DaemonSets.

Node Bootstrap

nodeadm Tool
The AL2023-era node initialization tool. Uses a YAML NodeConfig schema. Replaces bootstrap.sh. Runs as two systemd services: nodeadm-config (before user data) and nodeadm-run (after user data).
bootstrap.sh Tool
The legacy AL2 bootstrap script at /etc/eks/bootstrap.sh. Configures kubelet, connects node to cluster. Still used by AL2 and some custom AMIs.
NodeConfig
The YAML object (apiVersion: node.eks.aws/v1alpha1) passed to nodeadm. Minimum required fields: cluster name, API server endpoint, certificate authority, service CIDR.
User Data
The EC2 instance metadata script that runs at boot. For EKS nodes, this is where bootstrap commands or NodeConfig YAML goes. Specified in the launch template.

AWS Primitives (EKS-relevant)

VPC (Virtual Private Cloud) AWS
Your isolated network in AWS. An EKS cluster lives inside a VPC. Requires subnets in ≥2 AZs.
Subnet
A range of IPs in a single AZ. Private subnets (no internet gateway route) are for worker nodes. Public subnets are for load balancers.
IAM Role AWS
An AWS identity with permissions. EKS needs two: a cluster role (for the control plane) and a node role (for worker instances).
Launch Template AWS
An EC2 resource that defines instance configuration — AMI ID, instance type, user data, security groups, etc. Node groups reference a launch template.
Auto Scaling Group (ASG) AWS
Manages a fleet of EC2 instances — scales in/out, replaces unhealthy instances. Every node group is backed by an ASG.
CloudFormation AWS
AWS infrastructure-as-code service. eksctl generates CloudFormation stacks to create clusters and node groups.
AMI (Amazon Machine Image) AWS
A template for EC2 instances containing the OS, software, and configuration. EKS provides optimised AMIs; you can also bring your own.
ENI (Elastic Network Interface) AWS
A virtual network card attached to an EC2 instance. The VPC CNI plugin attaches ENIs to nodes and assigns secondary IPs to pods.

Tools

eksctl Tool
The official CLI for EKS. Creates clusters via CloudFormation. Supports YAML config files. Simplest way to get started.
aws eks Tool
The EKS subcommand of the AWS CLI. Lower-level than eksctl. Used for cluster creation, kubeconfig updates, and direct API calls.
kubectl K8s
Standard Kubernetes CLI. Works with EKS once kubeconfig is configured via aws eks update-kubeconfig.

Networking

VPC CNI Plugin AWS K8s
Assigns real VPC IP addresses to pods (no overlay network). Attaches ENIs to nodes, assigns secondary IPs from VPC CIDR to pods.
Security Group AWS
A virtual firewall for EC2 instances. EKS creates a cluster security group that allows control plane ↔ node communication.
NAT Gateway AWS
Allows instances in private subnets to reach the internet (for pulling images, etc.) without being directly reachable from the internet.
