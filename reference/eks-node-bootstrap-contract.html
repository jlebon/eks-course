EKS Node Bootstrap Contract

What an AMI needs to join an EKS cluster. Print-friendly reference.

1. Binaries Required on the AMI

BinaryPurposeReference path
kubeletNode agent — registers with API server, runs pods/usr/bin/kubelet
containerdContainer runtime/usr/bin/containerd
aws-iam-authenticatorIAM-based auth to the K8s API/usr/local/bin/aws-iam-authenticator
CNI pluginsPod networking/opt/cni/bin/
ecr-credential-providerECR image pull auth/etc/eks/image-credential-provider/
iptableskube-proxy networking rules/usr/sbin/iptables

2. Cluster Parameters (provided at boot)

ParameterNodeConfig fieldRetrieval command
Cluster namespec.cluster.nameKnown at deploy time
API server endpointspec.cluster.apiServerEndpointaws eks describe-cluster --name X --query cluster.endpoint
Certificate authorityspec.cluster.certificateAuthorityaws eks describe-cluster --name X --query cluster.certificateAuthority.data
Service CIDRspec.cluster.cidraws eks describe-cluster --name X --query cluster.kubernetesNetworkConfig.serviceIpv4Cidr

5. AWS Infrastructure Requirements

RequirementDetail
Node IAM rolePolicies: AmazonEKSWorkerNodePolicy, AmazonEKS_CNI_Policy, AmazonEC2ContainerRegistryReadOnly
Instance profileAttached to EC2 instance, references node IAM role
Security groupCluster SG — allows control plane ↔ node traffic
Node authorizationIAM role must be mapped to K8s identity via access entries or aws-auth ConfigMap
EC2 tagkubernetes.io/cluster/<name> = owned

6. nodeadm Boot Sequence (AL2023)

OrderServiceWhat happens
1nodeadm-configBefore user data — baseline containerd + kubelet config
2cloud-init user dataYour shell scripts run here
3nodeadm-runAfter user data — merges all config, starts kubelet, node registers

7. Config Files Written by nodeadm

FilePurpose
/etc/kubernetes/kubelet/config.jsonKubelet configuration (clusterDNS, auth, TLS bootstrap, runtime endpoint)
/var/lib/kubelet/kubeconfigKubelet kubeconfig (API server URL + auth)
/etc/eks/kubelet/environmentKubelet environment variables / startup flags
/etc/kubernetes/pki/ca.crtCluster CA certificate for TLS
/etc/containerd/config.tomlContainerd runtime configuration
/etc/eks/image-credential-provider/config.jsonECR credential provider config
/etc/sysctl.d/99-nodeadm.confSysctl settings (IP forwarding for pod networking)
/etc/modules-load.d/containerd.confKernel modules for container networking

8. Debugging Commands

# nodeadm boot logs
journalctl -u nodeadm-config --no-pager
journalctl -u nodeadm-run --no-pager

# Service status
systemctl status kubelet
systemctl status containerd

# Kubelet logs
journalctl -u kubelet -f

# Read user data from inside the node (IMDSv2)
TOKEN=$(curl -sX PUT "http://169.254.169.254/latest/api/token" \
  -H "X-aws-ec2-metadata-token-ttl-seconds: 60")
curl -s -H "X-aws-ec2-metadata-token: $TOKEN" \
  http://169.254.169.254/latest/user-data

# SSM session (from your workstation)
aws ssm start-session --target INSTANCE_ID
